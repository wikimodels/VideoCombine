# Технические Требования: Commando — Jungle Trench

Документ содержит полный перечень правил, механик и архитектурных принципов игры `CommandoScene`.

---

## 1. Архитектурный Принцип: Детерминизм и Пре-калькуляция (CRITICAL)

- **Запрет расчетов "на лету":** Любая логика внутри цикла render или в процессе анимации строго запрещена.
- **Пре-калькуляция:** Все пути перемещения, моменты выстрелов, траектории пуль и координаты взрывов рассчитываются заранее внутри хука `useMemo`.
- **База для расчетов:** Все игровые события (спавн врагов, выстрелы героя) жестко привязаны к анализу низких частот (bass frequencies) аудио-трека.
- **Результат:** Видео всегда рендерится одинаково кадр в кадр, независимо от производительности системы.

---

## 2. Система Волн и Отрядов (Wave System)

- **Состав отряда (Squad):** В каждой волне появляется **ровно 5 врагов**.
- **Разнообразие:** Обязательное присутствие как минимум по **одному врагу каждого типа** (`type_1`, `type_2`, `type_3`).
- **Условие новой волны:** Появление следующего отряда разрешено **только при условии полной зачистки** (смерти) всех 5 участников текущего отряда.

---

## 3. Правила Перемещения (Movement Policies)

- **Запрет на толкучку (No Crowding):** Враги должны быть распределены по оси Y при спавне. Наложение спрайтов минимизируется за счет разброса начальных точек.
- **Постоянное движение (No Freezing):** Персонажи никогда не стоят на месте. Враги движутся влево или патрулируют границу. Герой маневрирует, уклоняясь от пуль.
- **Плавность (No Jittering):** Пути рассчитываются как непрерывные последовательности координат. Микро-дергания, тряска и зависания в объектах запрещены.
- **Обход препятствий:** При встрече с объектом персонаж незамедлительно начинает обход вверх или вниз.

---

## 4. Территории и Границы (Territory Lockdown)

| Зона | X-диапазон |
|------|-----------|
| Зона Героя | 100 — 600 |
| Нейтральная зона | 601 — 1099 |
| Зона Врагов | 1100 — 1800 |

- **Жесткий лимит:** Враги физически не могут пересечь границу `1100` влево. Если враг достиг границы — остается на ней, продолжая движение по вертикали.

---

## 5. Объекты и Пули

- **Объекты:** Хранятся в массиве `worldObjects`. Блокируют персонажей и поглощают пули.
- **Пули героя:** Строго горизонтальные (Locked Y), продолговатые, желтые.
- **Пули врагов:** Короткие красные плазменные сгустки.
- **Попадание в объект:** Пуля исчезает, создавая скромный эффект искр (`spark`).
- **Точность:** 70% попаданий, 30% осечек (мимо).

---

## 6. Визуал "Вдребезги" (Ultra Shatter)

- **Моментальный разлет:** Спрайт исчезает в кадр хита.
- **Частицы:** Крупные блоки (20–30px), высокая плотность (до 45 фрагментов), реалистичный разлет с гравитацией.
- **Сегментация:** Цвет частиц зависит от части тела (шлем/торс) и типа врага.

---

## 7. Регламент Обновлений и Контроля (Governance)

- **Запрет односторонних изменений:** Любые изменения, удаления, упрощения или усложнения логики/кода без предварительного согласования с USER категорически запрещены.
- **Процедура верификации:** После каждого обновления кода Исполнитель обязан:
  1. Свериться с данным документом по каждому пункту.
  2. Предоставить отчет, где по каждому разделу **(1–9)** будет стоять отметка **Ок** (соблюдено).
- **Сохранение архитектуры:** Любые рефакторинги не должны нарушать принципы детерминизма и пре-калькуляции (Раздел 1).

---

## 8. Архитектура Скинов и Модульность (Skinning & Modularity)

- **Иммутабельность движка:** Математическое ядро (пре-калькуляция, законы движения, территория, логика волн) является неизменным.
- **Скинизация:** Весь визуальный слой (спрайты, скины объектов, формы пуль, ландшафт) отделен от логики.
- **Data-Driven подход:** Менять «картинку» игры возможно путем подмены конфигураций без вмешательства в алгоритмы Pass 1–4.
- Враги/Объекты/Снаряды подгружаются как внешние спрайты или SVG-конфигурации.
- Ландшафт задается отдельным слоем.

---

## 9. Конфигурация и Настройки (Settings)

- **Файл настроек:** `public/CommandoScene/CommandoScene.json`
- **Содержание:** JSON содержит конфигурацию сцены (`game`), размеры спрайтов, количество активов и координаты мушек (`muzzleOffsetX`, `muzzleOffsetY`).
- **Свойство `background`:** Указывает имя файла фона (например, `"background": "background.svg"`). Файл должен лежать в `public/{game}/`.
- **Логика Fallback:**
  - Если `game` пуст или файл не найден — движок использует встроенные SVG-скины и градиентный фон.
  - Если путь указан — движок приоритетно отрисовывает внешний SVG-ассет (или PNG/JPG фон).
- **Приоритет:** Бот создает игру «с нуля» при пустых настройках; пользователь мгновенно проводит рескин через конфиг.

## 10. Система Точки Выстрела (Muzzle Offset)

**Проблема:** при смене SVG-спрайта на кастомный, дуло оружия находится в другом месте — пуля будет вылетать из неверной точки.

**Решение — muzzle offset в конфиге:**
- В `CommandoScene.json` для героя и каждого типа врага (`type_1`, `type_2` и т.д.) указываются размеры спрайта (`width`, `height` общие) и смещение дула (`muzzleOffsetX`, `muzzleOffsetY`) в пикселях от левого верхнего угла спрайта.
- Движок рассчитывает абсолютную позицию дула формулой:
  ```javascript
  muzzleX = (spriteLeft) + muzzleOffsetX
  muzzleY = (spriteTop)  + muzzleOffsetY
  ```
- Эти координаты используются как источник пуль (Pass 4) и позиция вспышки (Muzzle Flash).

**Как замерить offset для своего SVG:**
1. Открыть SVG в Inkscape (или Figma).
2. Найти кончик ствола оружия.
3. Прочитать его координаты `X, Y` внутри SVG-viewport (ось Y должна смотреть вниз).
4. Записать в `CommandoScene.json`:
   ```json
    "hero": {
        "width": 120,
        "height": 140,
        "muzzleOffsetX": 72,
        "muzzleOffsetY": 53
    }
   ```
5. Аналогично для любого `type_N` объекта `enemies`. Больше ничего менять в TSX коде не нужно — движок и типы `EnemyType` подхватят новый ключ автоматически.

**Fallback:** если `skin` пуст, используется встроенный SVG, offset остаётся тем же (дефолтные значения совпадают с геометрией встроенного спрайта).

---

## 11. Вспышка от Выстрела (Muzzle Flash)

- **Триггер:** отображается в момент каждого выстрела героя (кадры из `commandoShots`).
- **Длительность:** **3 кадра** (`FLASH_DURATION = 3`).
- **Позиция:** строго на координатах дула (`muzzleX`, `muzzleY`) из Раздела 10.
- **Визуал:** SVG-вспышка типа «звезда» — 8 лучей, чередующихся по длине, с ярким белым ядром и жёлтым ореолом.
- **Анимация:** вспышка угасает (opacity и scale уменьшаются от 1 до 0 за 3 кадра).
- **Детерминизм:** вспышка рендерится чисто через `frame`, никакого внутреннего state не использует.

## 12. Конвенция Именования Ассетов (Asset Naming Convention)

**Принцип:** «Convention over configuration» — кидаешь файлы в папку с правильными именами, код подбирает всё сам.

### Структура папки

```
public/
└── {название_игры}/     ← имя задаётся в cfg.game в коде
    ├── hero.svg          → спрайт героя
    ├── enemy_1.svg       → тип 1 врага
    ├── enemy_2.svg       → тип 2 врага
    ├── enemy_3.svg       → тип 3 врага  (и т.д.)
    ├── object_1.svg      → crate
    ├── object_2.svg      → bush
    ├── object_3.svg      → tower
    ├── object_4.svg      → tree
    ├── object_5.svg      → stone  (и т.д.)
    ├── hero_bullet.svg   → пуля героя
    ├── enemy_bullet.svg  → пуля врага
    └── background.svg    → фон сцены
```

### Как управлять

Файл `public/CommandoScene/CommandoScene.json`:

```json
{
    "game": "CommandoScene",
    "hero": { "width": 120, "height": 140, "muzzleOffsetX": 72, "muzzleOffsetY": 53 },
    "enemies": {
        "count": 3,
        "width": 100, "height": 130,
        "type_1": { "muzzleOffsetX": 1, "muzzleOffsetY": 21 },
        "type_2": { "muzzleOffsetX": 2, "muzzleOffsetY": 16 },
        "type_3": { "muzzleOffsetX": 1, "muzzleOffsetY": 33 }
    },
    "objects": { "count": 2 },
    "background": "background.svg"
}
```

| Что сделать | Как |
|-------------|-----|
| Добавить нового врага (`type_4`) | Кинуть `enemy_4.svg`, добавить объект `"type_4": {...}` в JSON, поднять `enemies.count` до 4. Код автоматически расширит типы и волны. |
| Добавить объект | Кинуть `object_3.svg`, поднять `objects.count` до 3. Тип берётся из зашитого массива `OBJ_TYPES`. |
| Убрать кастомные скины | Очистить `game: ""` — движок вернётся к встроенным SVG. |
| Сменить фон | Указать `background: "file.jpg"` в JSON. |

### Фолбек (Fallback)

Если `game` пустой **или** индекс файла превышает `count` — компонент автоматически рисует встроенный SVG. Никаких ошибок рендеринга нет.

---

## 13. Анимация Ходьбы (Walking Animation)

**Не используется.** Персонажи представлены цельными статичными спрайтами (`hero.svg`, `enemy_N.svg`). Движение персонажей по полю обеспечивается системой путей (Раздел 3), а не спрайтовой анимацией. Никаких дополнительных эффектов поверх спрайта без согласования с USER.
